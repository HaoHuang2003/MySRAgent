# 符号回归常见错误（Gotchas）

本文档记录了符号回归任务中的常见错误和解决方案。

---

## 1. 公式格式错误

### ❌ 错误：使用 `^` 表示幂运算

```
evaluate_formula(eq="x1^2 + 2*x1 + 1")  # 错误！^ 在 Python 中是异或
```

### ✅ 正确：使用 `**` 表示幂运算

```
evaluate_formula(eq="x1**2 + 2*x1 + 1")  # 正确
```

**原因**：nd2py 解析器使用 Python 语法，`^` 是按位异或运算符。

---

## 2. 变量命名不匹配

### ❌ 错误：变量名与输入数据不一致

```python
# 输入数据：{"x1": [...], "x2": [...]}
evaluate_formula(eq="a*x + b")  # 错误！没有 x 这个变量
```

### ✅ 正确：使用输入数据中的特征名

```python
evaluate_formula(eq="a*x1 + b", x_vars=["x1"])  # 正确
```

**解决**：先用 `statistics_analysis` 查看可用的特征名称。

---

## 3. 忽略异常值

### ❌ 错误：直接拟合含异常值的数据

```
数据范围：[-100, 100]，但有一个点是 10000
→ 拟合结果会被异常值主导
```

### ✅ 正确：先检查数据分布

```
1. 调用 statistics_analysis
2. 检查 max/min 是否合理
3. 如有异常值，考虑：
   - 数据清洗
   - 使用稳健回归方法
   - 分段拟合
```

---

## 4. 过度复杂的公式

### ❌ 错误：追求完美拟合，使用过度复杂的表达式

```
eq = "sin(cos(exp(x1))) + tan(log(x2))"  # 过度嵌套，缺乏可解释性
```

### ✅ 正确：遵循奥卡姆剃刀原则

```
eq = "a*x1**2 + b*x1 + c"  # 简单且可解释
```

**判断标准**：
- R² > 0.99 即可，不必追求 1.0
- 公式深度不超过 3-4 层
- 参数有明确的物理/数学意义

---

## 5. 忽略量纲一致性

### ❌ 错误：量纲不匹配的运算

```
# x1 单位是米，x2 单位是秒
eq = "x1 + x2"  # 错误！米 + 秒 没有物理意义
```

### ✅ 正确：检查量纲

```
eq = "a*x1 + b"  # 正确，a 的单位是 1/米
```

---

## 6. 多项式阶数过高

### ❌ 错误：直接使用高阶多项式

```
polynomial_fit(max_degree=10)  # 严重过拟合
```

### ✅ 正确：从低阶开始

```
1. polynomial_fit(max_degree=2)
2. 检查残差
3. 如需要，增加到 max_degree=3 或 4
4. 关注 adjusted_r_squared，而不是 r_squared
```

**经验法则**：
- 样本数 n < 50：max_degree ≤ 2
- 样本数 n ≥ 100：max_degree ≤ 4
- 始终检查 adjusted_r_squared

---

## 7. 不考虑交互项约束

### ❌ 错误：允许所有变量交互

```
# 5 个变量，允许所有交互 → C(5,2) = 10 个交互项
→ 模型复杂度过高
```

### ✅ 正确：使用黑名单/白名单

```
polynomial_fit(
    max_degree=2,
    interaction_blacklist=[("x1", "x2")]  # 不允许 x1 和 x2 交互
)
```

---

## 8. 忽略参数优化

### ❌ 错误：手动调整系数

```
eq = "2.5*x1 + 3.7*x2 + 1.2"  # 手动试系数
```

### ✅ 正确：使用自动拟合

```
evaluate_formula(eq="a*x1 + b*x2 + c", fit=True)
# BFGS 算法会自动优化 a, b, c
```

---

## 9. 残差不分析

### ❌ 错误：只看 R²，不检查残差

```
R² = 0.95 → 认为拟合很好 → 结束
# 但残差可能有系统性偏差
```

### ✅ 正确：分析残差模式

```
1. evaluate_formula 返回 y_pred
2. 计算 residual = y_true - y_pred
3. 观察残差：
   - 随机分布 → 拟合良好
   - U 型曲线 → 缺少二次项
   - 周期性 → 缺少三角函数项
```

---

## 10. 缺少迭代思维

### ❌ 错误：一次拟合就结束

```
polynomial_fit(...) → R²=0.85 → 放弃
```

### ✅ 正确：迭代优化

```
1. 基础拟合 → R²=0.85
2. 分析残差 → 发现周期性模式
3. 添加三角函数项 → evaluate_formula(eq="a*x + b*sin(c*x)")
4. 参数优化 → fit=True
5. 新 R²=0.98 → 成功
```

**迭代流程**：
```
数据探索 → 基础拟合 → 残差分析 → 提出假设 → 验证 → 重复
```

---

## 总结检查清单

在提交最终公式前，检查：

- [ ] 公式使用 `**` 而非 `^`
- [ ] 变量名与输入数据匹配
- [ ] R² > 0.99（或达到任务要求）
- [ ] adjusted_r_squared 接近 r_squared
- [ ] 残差无明显系统性模式
- [ ] 公式简洁，深度 ≤ 4
- [ ] 系数量级合理
- [ ] 量纲一致（如适用）
